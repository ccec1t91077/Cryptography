# Password Encryption Tool

Encrypt a plain-text password using classical cipher algorithms to generate a reproducible strong password for website login.

[繁體中文](../README.md)

---

## Features

- Supports seven classical cipher rules, freely combinable and reorderable
- All operations run locally in the browser — no data is transmitted
- Preferences (rule parameters, symbol options, password length, etc.) are automatically saved to `localStorage`
- Fully static page with no backend — open directly as a file or deploy to any static host
- Supports switching between Traditional Chinese and English interfaces

---

## How to Use

1. In the **Encryption Rules** panel on the right, click **Add Rule**, choose a cipher, and fill in its parameters. Multiple rules can be added and applied in sequence.
2. Enter the plain-text **Password** you keep in your notes.
3. Optionally provide a **Salt**, and adjust the special characters, password length, and auto-fill options as needed.
4. Click **Generate Password** and copy the result for login.

> The same inputs and settings always produce the same result — you can regenerate your password at any time.

## Cipher Rules

| Name | Parameters | Description |
|------|------------|-------------|
| Caesar Cipher | Shift (can be negative) | Shifts each character a fixed number of positions along the character set |
| Affine Cipher | Multiplier, Shift | Applies `(a×x + b) mod m` to each character for a double transformation |
| Vigenère Cipher | Key string | Uses a repeating key to apply a different shift to each character, avoiding patterns |
| Playfair Cipher | Keyword | Builds a lookup table from a keyword and encrypts characters in pairs |
| Hill Cipher | 2×2 key matrix (a/b/c/d) | Mixes pairs of characters through matrix multiplication |
| Rail Fence Cipher | Rails (1–24) | Distributes characters in a zigzag pattern across rails, then reads them off rail by rail |
| Columnar Transposition | Key string | Fills characters into a grid and reads columns in the order determined by sorting the key letters |

## Encryption Flow

```
Input: password + salt
  ↓
Apply cipher rules in order (top to bottom)
  ↓
Length adjustment (excess folds into the base via index XOR; shortage loops to fill)
  ↓
Auto-fill character types (ensures uppercase, lowercase, digits, special characters)
  ↓
Output encrypted password
```

## Options

**Special Characters**
Check the symbols allowed in the output (`! @ # $ % ^ & * - _ + = . ?`). Unchecked symbols are excluded from the character set and affect all cipher calculations.

**Password Length**
Range 6–24, default 8. Excess characters are folded into the base via index XOR to preserve information.

**Auto-fill Character Types**
When enabled, if the generated password is missing a character type, a deterministic hash selects a position and character to replace, ensuring the result meets common password requirements.

## Data Storage

Preferences are stored in `localStorage` with a key derived from a hash of the current page's `origin + pathname`, preventing cross-path conflicts.

| Path | Content |
|------|---------|
| `prefs.symbols` | Selected special characters |
| `prefs.length` | Password length |
| `prefs.showPassword` | Password visibility state |
| `prefs.autoFill` | Auto-fill toggles |
| `prefs.lang` | Interface language |
| `prefs.darkMode` | Dark mode state |
| `prefs.ruleList` | Rule list (type and parameters) |
| `flags.firstUseSeen` | Whether the first-use notice has been seen |

To clear, use your browser's "Clear site data" feature.

## Technical Details

- Pure HTML / CSS / JavaScript — no external dependencies or frameworks
- Fonts are loaded from the local `fonts/` directory — fully offline capable
- The Affine cipher's multiplier is restricted to values coprime with the character set length, dynamically generated by `populateSelect()`
- The Hill cipher validates that the key matrix determinant is invertible mod m before encrypting
- The Playfair cipher supports any character set size and automatically calculates the nearest square grid dimensions

## Security Notes

### Purpose

This tool is designed to **convert a memorable plain-text password into a hard-to-guess encrypted password**, suitable for:

- Generating unique passwords for different websites from the same plain-text password
- Regenerating passwords in untrusted environments without a password manager
- As a supplement to a password manager

### Security Considerations

**Advantages**
- All operations run locally — no data is transmitted
- Reproducible: regenerate anytime from the same inputs
- No need to store the encrypted password

**Limitations**
- Classical ciphers are not suitable for modern cryptographic requirements
- If the plain-text password or rule parameters are guessed, the encrypted password can be reproduced
- For critical accounts, a dedicated password manager is recommended
