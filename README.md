# 密碼加密工具

將明文密碼透過古典密碼學演算法加密，產生可重現的強密碼，用於網站登入。

> 📖 [English Documentation](docs/README.en.md)

---

## 功能概覽

- 支援七種古典加密規則，可自由新增、排序與組合，依序套用
- 所有運算在瀏覽器本地執行，不傳輸任何資料
- 偏好設定（規則參數、符號選項、密碼長度等）自動儲存至 `localStorage`
- 純靜態頁面，無後端伺服器，可直接以檔案開啟或部署至靜態主機
- 支援繁體中文與英文介面切換

---

## 使用方式

1. 在右側**加密規則**面板點擊「新增規則」，選擇加密方式並填入參數，可新增多條規則依序套用
2. 輸入記在備忘錄的明文 **Password**
3. 視需要填入 **Salt**、調整特殊符號、密碼長度與自動補充字元選項
4. 點擊「產生登入用密碼」，複製結果用於登入

> 相同的輸入與設定每次都會產出相同結果，可隨時重新產生。

## 加密規則

| 名稱 | 參數 | 說明 |
|------|------|------|
| 凱薩密碼 | 位移量（可為負數） | 將每個字元在字元表上往後移動固定距離 |
| 仿射密碼 | 乘數、位移量 | 以 `(a×x + b) mod m` 對每個字元進行雙重混淆 |
| 維吉尼亞密碼 | 金鑰字串 | 以金鑰循環決定每個字元的位移量，避免規律性 |
| 波雷費密碼 | 關鍵字 | 以關鍵字建立字元對照表，兩字元一組進行替換 |
| 希爾密碼 | 2×2 金鑰矩陣（a/b/c/d） | 兩字元一組進行矩陣乘法混合運算 |
| 柵欄密碼 | 軌道數（1–24） | 將字元按鋸齒形分配到軌道上，再依軌道順序讀出 |
| 列移位密碼 | 金鑰字串 | 將字元填入表格，依金鑰字母排序決定欄的讀取順序 |

## 加密流程

```
輸入：password + salt
  ↓
依規則列表由上至下依序套用加密規則
  ↓
長度處理（超出部分與前段做 index XOR 折疊；不足則循環補齊）
  ↓
自動補充字元（確保包含大寫、小寫、數字、特殊符號）
  ↓
輸出加密密碼
```

---

## 選項說明

**特殊符號**
勾選允許出現在加密結果中的符號（`! @ # $ % ^ & * - _ + = . ?`），取消勾選的符號將從字元表中排除，影響所有加密規則的運算結果。

**密碼長度**
範圍 6–24，預設 8。超出目標長度的部分會與前段做 index XOR 折疊，確保資訊不丟失。

**自動補充字元**
勾選後，若產生的密碼缺少該類型字元，會以確定性 hash 選定位置與字元進行替換，確保結果符合常見密碼規則。

## 資料儲存

偏好設定以 `localStorage` 儲存，key 由當前頁面的 `origin + pathname` 雜湊產生，避免跨路徑衝突。

| 路徑 | 內容 |
|------|------|
| `prefs.symbols` | 已勾選的特殊符號清單 |
| `prefs.length` | 密碼長度 |
| `prefs.showPassword` | 明文密碼顯示狀態 |
| `prefs.autoFill` | 自動補充字元各項開關 |
| `prefs.lang` | 介面語言 |
| `prefs.darkMode` | 暗色模式狀態 |
| `prefs.ruleList` | 規則列表（類型與參數） |
| `flags.firstUseSeen` | 是否已看過首次使用說明 |

如需清除，請透過瀏覽器的「清除網站資料」功能。

## 技術細節

- 純原生 HTML / CSS / JavaScript，無任何外部依賴與框架
- 字型透過本地 `fonts/` 目錄載入，完全離線可用
- 仿射密碼的乘數限定為與字元表長度互質的值，由 `populateSelect()` 動態產生選單
- 希爾密碼在執行前會驗證金鑰矩陣的行列式是否在模 m 下可逆
- 波雷費密碼支援任意字元表大小，自動計算最接近正方形的矩陣維度

## 安全性說明

### 本工具的定位

此工具設計用於**將易記的明文密碼轉換為難猜的加密密碼**，適合以下場景：

- 使用相同明文密碼為不同網站產生獨立密碼
- 在不信任的環境重新產生密碼，無需攜帶密碼管理器
- 作為密碼管理器的補充方案

### 安全性考量

**優點**
- 所有運算在本地執行，不傳輸任何資料
- 可重現性：只要記住明文密碼，隨時可重新產生
- 無需儲存加密後的密碼，降低洩漏風險

**限制**
- 古典密碼學演算法不適合直接用於現代加密需求
- 若明文密碼或規則參數被猜測，加密密碼可被重現
- 重要帳號建議使用專業密碼管理器
