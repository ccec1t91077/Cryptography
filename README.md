# 密碼加密工具

將明文密碼透過古典密碼學演算法加密，產生可重現的強密碼，用於網站登入。

## 功能概覽

- 支援五種古典加密規則，可透過 Plan Code 自由組合、依序套用
- 所有運算在瀏覽器本地執行，不傳輸任何資料
- 偏好設定（規則參數、符號選項、密碼長度等）自動儲存至 `localStorage`
- 純靜態頁面，無後端伺服器，可直接以檔案開啟或部署至靜態主機

## 使用方式

1. 在右側**加密規則**面板填入各規則所需的參數
2. 輸入 **Plan Code**，其中的數字對應規則編號，可組合文字與數字，例如 `gmail131` 代表依序套用凱薩 → 維吉尼亞 → 凱薩
3. 輸入記在備忘錄的明文 **Password**
4. 視需要填入 **Salt**、調整特殊符號、密碼長度與自動補充字元選項
5. 點擊「產生登入用密碼」，複製結果用於登入

> 相同的輸入與設定每次都會產出相同結果，可隨時重新產生。

## 加密規則

| 編號 | 名稱 | 參數 | 說明 |
|------|------|------|------|
| 1 | 凱薩密碼 | 位移量（可為負數） | 將每個字元在字元表上往後移動固定距離 |
| 2 | 仿射密碼 | 乘數 a、位移量 b | 以 `(a×x + b) mod m` 對每個字元進行雙重混淆 |
| 3 | 維吉尼亞密碼 | 金鑰字串 | 以金鑰循環決定每個字元的位移量，避免規律性 |
| 4 | 波雷費密碼 | 關鍵字 | 以關鍵字建立字元對照表，兩字元一組進行替換 |
| 5 | 希爾密碼 | 2×2 金鑰矩陣（a/b/c/d） | 兩字元一組進行矩陣乘法混合運算 |

### Plan Code 組合範例

| Plan Code | 套用順序 |
|-----------|----------|
| `131` | 凱薩 → 維吉尼亞 → 凱薩 |
| `google2` | 仿射 |
| `fb13` | 凱薩 → 維吉尼亞 |

## 加密流程

```
輸入：password + planCode + salt
  ↓
依 Plan Code 中的數字依序套用加密規則
  ↓
長度處理（超出部分與前段做 index XOR 折疊；不足則循環補齊）
  ↓
自動補充字元（確保包含大寫、小寫、數字、特殊符號）
  ↓
輸出加密密碼
```

## 選項說明

**特殊符號**
勾選允許出現在加密結果中的符號（`! @ # $ % ^ & * - _ + = . ?`），取消勾選的符號將從字元表中排除，影響所有加密規則的運算結果。

**密碼長度**
範圍 6–24，預設 8。超出目標長度的部分會與前段做 index XOR 折疊，確保資訊不丟失。

**自動補充字元**
勾選後，若產生的密碼缺少該類型字元（大寫、小寫、數字、特殊符號），會以確定性 hash 選定位置與字元進行替換，確保結果符合常見密碼規則。

## 資料儲存

偏好設定以 `localStorage` 儲存，key 由當前頁面的 `origin + pathname` 雜湊產生，避免跨路徑衝突。儲存項目包含：

| 路徑 | 內容 |
|------|------|
| `prefs.symbols` | 已勾選的特殊符號清單 |
| `prefs.length` | 密碼長度 |
| `prefs.showPassword` | 明文密碼顯示狀態 |
| `prefs.autoFill` | 自動補充字元各項開關 |
| `prefs.caesar.shift` | 凱薩位移量 |
| `prefs.affine.a` / `prefs.affine.b` | 仿射乘數與位移量 |
| `prefs.vigenere.key` | 維吉尼亞金鑰 |
| `prefs.playfair.key` | 波雷費關鍵字 |
| `prefs.hill` | 希爾矩陣（a/b/c/d） |
| `session.planCode` | Plan Code |
| `flags.firstUseSeen` | 是否已看過首次使用說明 |

如需清除，請透過瀏覽器的「清除網站資料」功能。

## 技術細節

- 純原生 HTML / CSS / JavaScript，無任何外部依賴與框架
- 字型透過本地 `fonts/` 目錄載入，完全離線可用
- 仿射密碼的乘數 a 限定為與字元表長度互質的值，由 `populateSelect()` 動態產生選單
- 希爾密碼在執行前會驗證金鑰矩陣的行列式是否在模 m 下可逆
- 波雷費密碼支援任意字元表大小，自動計算最接近正方形的矩陣維度
- `details` 展開動畫使用 CSS `grid-template-rows: 0fr → 1fr` 過渡實現，不依賴 JavaScript 計算高度


## 安全性說明

### 本工具的定位

此工具設計用於**將易記的明文密碼轉換為難猜的加密密碼**，適合以下場景：

- 使用相同明文密碼 + 不同 Plan Code 為不同網站產生獨立密碼
- 在不信任的環境（如網咖、公用電腦）重新產生密碼，無需攜帶密碼管理器
- 作為密碼管理器的補充方案，用於不便使用密碼管理器的場合

### 安全性考量

**優點：**
- 所有運算在本地執行，不傳輸任何資料
- 可重現性：只要記住明文密碼與 Plan Code，隨時可重新產生
- 無需儲存加密後的密碼，降低洩漏風險

**限制：**
- 古典密碼學演算法不適合直接用於現代加密需求
- 若明文密碼或 Plan Code 被猜測，加密密碼可被重現
- 不提供密碼強度評估，需使用者自行確保明文密碼的複雜度

**建議：**
- 明文密碼應具備一定複雜度（長度、字元多樣性）
- Plan Code 應包含個人化資訊（如網站名稱），避免跨站重複
- 重要帳號（如銀行、Email）建議使用專業密碼管理器

## 開發說明

### 程式碼架構

**模組化設計（IIFE 模式）：**
- `StorageManager` — localStorage 讀寫與資料持久化
- `FirstUseNotice` — 首次使用說明彈窗
- `PasswordVisibility` — 明文密碼顯示/隱藏切換
- `CharsetManager` — 字元表管理（大寫、小寫、數字、特殊符號）
- `CaesarCipher` — 凱薩密碼加密
- `AffineCipher` — 仿射密碼加密與乘數選單產生
- `VigenereCipher` — 維吉尼亞密碼加密
- `PlayfairCipher` — 波雷費密碼加密（支援任意字元表）
- `HillCipher` — 希爾密碼加密與金鑰驗證
- `EncryptionManager` — 統籌加密流程與結果處理

### 關鍵實作細節

**字元表動態性：**
所有加密演算法接受 `charset` 參數，由 `CharsetManager.get()` 根據使用者勾選的特殊符號動態產生。當符號選項改變時，仿射密碼的乘數選單會自動重新計算。

**確定性自動補充：**
自動補充字元使用確定性 hash（`password + planCode + salt + type`）選定替換位置與字元，確保相同輸入產生相同結果。同時保護「唯一字元」位置（如結果中僅有一個數字時，該位置不會被替換為其他類型）。

**長度處理的 XOR 折疊：**
當加密結果超出目標長度時，超出部分與前段做 index XOR（在字元表內做索引運算），確保所有資訊都影響最終結果，避免單純截斷造成的資訊丟失。

**波雷費矩陣維度計算：**
傳統波雷費密碼使用 5×5 矩陣（26 字母去除 J），本實作支援任意字元表大小，自動計算最接近正方形的 `rows × cols` 維度（`rows ≤ cols`）。

**希爾密碼可逆性驗證：**
在執行希爾密碼前，會計算金鑰矩陣的行列式並檢查其在模 m 下是否存在乘法反元素，若不可逆則提示使用者調整數值。

- 採用 Apple Human Interface Guidelines 風格
- CSS 變數集中管理顏色與尺寸
- 響應式設計：螢幕寬度 ≤ 900px 時切換為單欄佈局
- 自訂 `<select>` 元件（`.custom-select`）提供更好的視覺一致性

- 需支援 ES6+ 語法（箭頭函式、解構、模板字串、`const`/`let`）
- 需支援 CSS Grid、CSS 變數、`:has()` 選擇器
